Screenshot 3 · Global reach

Switched to Tokyo in one click

Same session, same UI — different city. No regional hardcoding, no geo-lock. The cascading location picker queries Open-Meteo's free geocoding API, so any city on Earth works.

Proof of global reach:
  • Tokyo forecast pulled live: 19°C, feels like 18°C, 41% humidity, partly cloudy
  • 7-day chart updated to show rain arriving Thursday with 69% precipitation probability
  • Location label correctly shows Tokyo · Japan
  • No API keys needed — works for any coordinates on Earth via Open-Meteo
Screenshot 4 · Detail

Ensemble confidence and per-provider weights

The fusion engine shows exactly how each provider contributes to the final prediction. When more providers go live, you'll see genuine multi-model disagreement — that's the entire point.

Screenshot 5 · CV pipeline

Nearby cameras — legally-sourced only

The computer-vision module is hooked up and ready. In v0.1 it gracefully handles the "no Windy API key" state; drop a key into .env and thumbnails populate instantly, each labeled by the heuristic classifier (HSV + Sobel) or a fine-tuned EfficientNet-B0.

Legal-first camera sources only:
  • Windy Webcams API v3 — official ToS-compliant integration
  • US DOT 511 state feeds — public-domain highway cameras
  • NOAA / National Weather Service webcams — federal open data
  • Explicitly excludes Insecam, Opentopia, and any unsecured default-password cameras (see LEGAL.md)
